# Zootopia Club Next Architecture

## Runtime Ownership

- `C:\zootopia-club-next` is the monorepo root and owns workspace/package-manager orchestration, shared lint/type/build tooling, docs, and shared package coordination.
- `apps/web` owns the canonical Next.js App Router runtime, theme/locale providers, Auth.js integration, Supabase adapters, and live route-handler backend.
- `apps/api` is a future extraction boundary and does not receive production traffic.
- `packages/shared-*` hold contracts, pure helpers, and configuration.

## Current Truth

- Active architecture stack: Supabase Auth + Supabase Postgres + Supabase Storage + Auth.js session cookies + Vercel deployment.
- `apps/web` is the only live Next.js runtime.
- `apps/web/app/api/**/route.ts` is the live backend surface for auth, uploads, assessment, infographic, and admin operations.
- `proxy.ts` is optimistic redirect support only; server layouts and route handlers remain authoritative gates.
- `apps/api` stays intentionally idle in the current runtime shape.

## Deployment Paths

### Local Development

- `apps/web` runs the UI and same-origin route handlers.
- `apps/api` remains inactive unless intentionally developed.
- Supabase service-role and database credentials remain server-only.

### Vercel (Active)

- Active and canonical deployment target.
- `vercel.json` points the runtime root to `apps/web`.
- Browser and server env values are managed through Vercel project environment variables.

### Legacy/Inactive Deployment References

- Firebase App Hosting and Firebase Hosting + Cloud Run are historical only.
- `apps/web/apphosting.yaml` is retained for legacy rollback/reference and does not define active deployment truth.
- `docs/legacy-reference/*` is archive context, not active runtime guidance.

## Data and Storage Targets

- Identity provider: Supabase Auth.
- Database source of truth: Supabase Postgres (`SUPABASE_DATABASE_URL` / `DATABASE_URL`) with schema managed in `supabase/migrations/*.sql`.
- Object storage source of truth: Supabase Storage (`zootopia-private`) with owner-scoped paths.
- Session layer: Auth.js JWT session cookies (`zc_session`) enforced server-side.

## Historical Notes

- Firebase, Firestore, Firebase Storage, Firebase Auth, Firebase Hosting, and Firebase App Hosting references are historical-only context unless a section explicitly says legacy/reference-only.
