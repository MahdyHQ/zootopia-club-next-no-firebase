# Legacy Reference Material

This folder contains the previous Firebase Hosting + Cloud Run configuration inputs that were moved out of the active project root during the monorepo migration.

These files are reference-only:

- they do not define the active deployment path
- they should not drive new implementation decisions
- any secrets or environment-specific values must remain local-only

Active deployment/runtime truth is Vercel + Supabase + Auth.js.
The `/firebase` folder exists for legacy Firebase-era rules/index artifacts and historical rollback context only.
